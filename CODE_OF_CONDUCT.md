# Code of Conduct

## Our Standards

People participating in the Hyperpipe project are expected to:

- communicate respectfully and assume good faith
- focus criticism on ideas, code, and behavior rather than people
- avoid harassment, discrimination, intimidation, and personal attacks
- avoid publishing private, sensitive, or doxxing material
- respect project boundaries, review scope, and maintainer decisions

Unacceptable behavior includes:

- harassment or abusive language
- sexualized language or unwelcome sexual attention
- threats, stalking, or sustained disruption
- trolling, dogpiling, or bad-faith escalation
- publishing someone else's private information without permission

## Scope

This code of conduct applies in:

- GitHub issues, pull requests, reviews, and discussions
- project chat or coordination channels run by the maintainers
- any public or private spaces where a person is representing the project

## Reporting

Do not use a public issue for conduct reports.

Report serious conduct concerns privately to the repository owner through GitHub or
another private contact channel already established with the maintainers.

## Enforcement

Maintainers may remove comments, close threads, reject contributions, or temporarily
or permanently restrict participation when behavior violates this policy.

Enforcement decisions are made to protect the project and its contributors, not to
win arguments.
